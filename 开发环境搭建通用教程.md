# RK2206 ／ RV1106 设备开发环境搭建通用教程

# 🚀 嵌入式开发环境搭建指南 (WSL2 + Docker)

　　针对 **RK2206 (OpenHarmony)**  与 **RV1106** 开发板，解决 Windows 无法直接编译的问题。

## 📌 核心思路

　　**Windows 不能直接编译** $\rightarrow$ 借助 **WSL2 (Ubuntu)**  作为 Linux 宿主 $\rightarrow$ 在其中安装 **Docker** 提供隔离的编译容器。

---

## 🛠️ 第一阶段：基础环境配置

### 1. 安装 WSL2 (Ubuntu 20.04)

　　以管理员身份打开 PowerShell，执行：

```PowerShell
wsl --install -d Ubuntu-20.04
```

> **注：**  此命令会自动启用虚拟化功能并安装内核。安装后执行 `wsl -l -v`​ 确保版本为 ​**2**。

### 2. 配置 WSL2 资源限制

　　在 Windows 用户目录（`C:\Users\你的用户名\`​）下创建 `.wslconfig` 文件，优化编译性能：

```
[wsl2]
processors=8       # 逻辑处理器数量，建议设为实际核心数
memory=16GB        # 建议设为物理内存的 50%
swap=32GB          # 编译大项目（如 OpenHarmony）建议设大
guiApplications=false
```

　　*修改后需在 PowerShell 执行* *​`wsl --shutdown`​*​ *重启生效。*

## 🐧 第二阶段：Ubuntu 内部配置

### 3. 系统环境调整

　　进入 WSL（输入 `wsl`），执行以下命令：

- **切换 Shell 为 Bash：**
- **一键换源（推荐）：**
- ```Bash
  curl -sSL https://linuxmirrors.cn/main.sh | sudo bash
  sudo apt update
  sudo apt install build-essential git curl wget vim -y
  ```

### 4. 安装 Docker Engine

　　在 WSL2 中建议安装原生 Docker 性能更佳：

```Bash
# 1. 预装依赖
sudo apt-get update && sudo apt-get install -y curl ca-certificates

# 2. 一键安装 Docker 并换源 (中科大)
curl -sSL https://linuxmirrors.cn/docker.sh | sudo bash -s -- --source "mirrors.ustc.edu.cn/docker-ce"

# 3. 配置镜像加速器
curl -sSL https://linuxmirrors.cn/docker.sh | sudo bash -s -- --only-registry --source-registry "docker.1ms.run,docker.m.daocloud.io"

# 4. 启动服务
sudo service docker start
```

### 6. 搭建 Docker 编译环境

#### 方案 A：RK2206 (使用官方镜像)

　　Bash

```
# 拉取镜像
docker pull swr.cn-south-1.myhuaweicloud.com/openharmony-docker/docker_oh_mini:3.2

# 创建并挂载容器 (替换“你的用户名”)
docker run -d --name oh_compiler \
  -v /home/你的用户名/work/rk2206:/home/openharmony/work \
  swr.cn-south-1.myhuaweicloud.com/openharmony-docker/docker_oh_mini:3.2 tail -f /dev/null
```

#### 方案 B：RV1106 (自定义 Dockerfile)

　　在工作目录创建 `docker-compose.yml`​ 配合 `Dockerfile`，集成交叉编译工具链：

　　YAML

```
services:
  rk1106-dev:
    build: .
    container_name: lockzhiner_dev
    volumes:
      - .:/workspace
    tty: true
    stdin_open: true
```

## 🏗️ 第四阶段：进入容器编译

### 7. 编译流程详解

|**开发板**|**进入容器命令**|**关键编译步骤**|
| --| ------| --------------------|
|**RK2206**|​`docker exec -it oh_compiler /bin/bash`|​`hb set`(选 rk2206)$\rightarrow$ `hb build -f`|
|**RV1106**|​`docker exec -it lockzhiner_dev bash`|​`cmake -B build ...` $\rightarrow$ `cmake --build build`|
