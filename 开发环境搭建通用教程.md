# RV1106 开发环境搭建全指南 (WSL2 + Docker)

本项目旨在为 **RV1106** 开发板提供一套高效、稳定的开发环境。通过 Windows 的 **WSL2** 子系统结合 **Docker** 容器技术，解决跨平台编译的复杂性。

---

## 🛠️ 第一阶段：Windows 基础环境配置

### 1. 安装并配置 WSL2 (Ubuntu 22.04)

WSL2 是在 Windows 上运行 Linux 的最佳方式。建议使用 Ubuntu 22.04 以获得最佳兼容性。

- **安装命令**：以管理员身份打开 PowerShell，执行：
  ```PowerShell
  wsl --install -d Ubuntu-22.04
  ```
- **版本检查**：安装完成后，执行 `wsl -l -v` 确保版本号为 **2**。

### 2. 优化 WSL2 资源管理

在 Windows 用户目录（`C:\Users\你的用户名\`）下创建或编辑 `.wslconfig` 文件，防止 WSL2 占用过多物理内存：

```ini
[wsl2]
processors=8       # 建议设为逻辑核心数
memory=8GB         # 建议设为物理内存的 50%
swap=16GB          # 虚拟内存
guiApplications=false
```
*修改后需在 PowerShell 执行 `wsl --shutdown` 重启生效。*

## 🐧 第二阶段：Linux 系统与 Docker 配置

进入 WSL（在终端输入 `wsl`），按照以下步骤配置：

### 3. 系统换源与基础工具安装

为了提高下载速度，推荐使用 [LinuxMirrors](https://linuxmirrors.cn/) 一键脚本。你可以根据网络情况自主选择 **中科大 (ustc)** 或 **阿里云 (aliyun)**。

```bash
# 运行一键换源脚本
curl -sSL https://linuxmirrors.cn/main.sh | sudo bash

# 安装基础开发工具
sudo apt update && sudo apt install build-essential git curl wget vim -y
```

### 4. Docker 安装与镜像加速

在 WSL2 中安装原生 Docker 引擎，并配置国内镜像加速器以解决镜像拉取慢的问题。

- **一键安装 Docker**：
  ```bash
  # 使用脚本安装，可根据提示选择镜像源（如中科大、阿里云）
  curl -sSL https://linuxmirrors.cn/docker.sh | sudo bash
  ```

- **配置镜像加速器**（如果安装脚本未自动配置）：
  ```bash
  # 再次运行脚本，仅配置镜像加速
  curl -sSL https://linuxmirrors.cn/docker.sh | sudo bash -s -- --only-registry --source-registry "docker.1ms.run,docker.m.daocloud.io"
  
  # 启动 Docker 服务
  sudo service docker start
  ```

- **权限设置**：为了避免每次都输入 `sudo`，将当前用户加入 docker 组：
  ```bash
  sudo usermod -aG docker $USER
  # 注意：执行后需要重启 WSL (wsl --shutdown) 才能生效
  ```

## 🏗️ 第三阶段：RV1106 开发环境实战

### 5. 获取项目与初始化

**重要提示**：务必将代码存放在 WSL 内部路径（如 `~/workspace`），**不要**放在 Windows 挂载路径（`/mnt/c/...`），否则编译速度会慢 10 倍以上。

```bash
git clone https://github.com/kaouduo/rv1106_vision_work.git
cd rv1106_vision_work

# 运行项目初始化脚本 (下载第三方库)
chmod +x setup_rv1106.sh
./setup_rv1106.sh
```

### 6. 容器管理与编译技巧

使用 Docker Compose 快速管理你的开发环境。

- **启动并进入环境**：
  ```bash
  docker compose up -d --build  # 后台启动并构建
  docker exec -it rv1106_dev bash  # 进入交互式终端
  ```

- **现代 CMake 编译技巧**：
  进入容器后，推荐使用以下流程进行编译，既清晰又支持多核加速：
  ```bash
  cd Cpp_example/A01_helloworld
  cmake -B build -DCMAKE_TOOLCHAIN_FILE=../../toolchains/arm-rockchip830-linux-uclibcgnueabihf.toolchain.cmake
  cmake --build build --parallel  # 自动调用所有 CPU 核心
  ```

## 📑 WSL2 与 Docker 使用管理技巧

### 1. 文件互操作
- **Windows 访问 WSL 文件**：在资源管理器地址栏输入 `\\wsl.localhost\Ubuntu-22.04\`。
- **WSL 访问 Windows 文件**：路径位于 `/mnt/c/Users/...`。

### 2. Docker 常用管理
| 操作 | 命令 | 说明 |
| :--- | :--- | :--- |
| **查看运行状态** | `docker ps` | 查看当前正在运行的容器 |
| **查看所有容器** | `docker ps -a` | 包含已停止的容器 |
| **查看镜像列表** | `docker images` | 查看本地已下载/构建的镜像 |
| **清理虚悬镜像** | `docker image prune` | 删除标签为 `<none>` 的过期镜像 |
| **彻底移除环境** | `docker compose down` | 停止并删除容器、网络 |

### 3. 常见问题排查
- **Docker 无法启动**：在 WSL 中运行 `sudo service docker status`。如果未启动，运行 `sudo service docker start`。
- **权限拒绝 (EACCES)**：如果在主机修改了容器生成的文件报错，请检查文件所有者，必要时使用 `sudo chown -R $USER:$USER .` 修复权限。
